# Spec: Party Roster Cleanup & Team Template Repurposing

**Created:** 2026-03-24
**Source:** BMAD Method v6.2.1 — "Removed dead party roster files"
**Backlog refs:** I4 (convention alignment), P14 (Team Factory)

## Overview

### Problem Statement

BMAD Method v6.2.1 removed party roster files because party-mode now loads agents exclusively from `agent-manifest.csv`. Our project has 3 `default-party.csv` files (`bmad/`, `tea/`, `cis/` modules) that are **not read by party-mode at runtime** — party-mode reads from `_bmad/_config/agent-manifest.csv`. These files are dead code that could confuse contributors into thinking they affect agent selection.

### Solution

Remove `default-party.csv` files to align with upstream. Optionally, evaluate whether the Team Factory (P14) could use a "team template" CSV concept for scaffolding — if so, design that format separately rather than repurposing dead files.

### Scope

**In Scope:**
- Verify zero runtime references to `default-party.csv` across all code and workflows
- Remove 3 `default-party.csv` files
- Verify party-mode still works correctly (reads from `agent-manifest.csv`)
- Document decision: Team Factory team templates (if needed) will be a separate design

**Out of Scope:**
- Changing party-mode's agent loading mechanism
- Designing Team Factory team template format (future P14 scope)
- Modifying `agent-manifest.csv` structure

## Context for Development

### Codebase Patterns

Party-mode's `step-01-agent-loading.md` reads from `{project-root}/_bmad/_config/agent-manifest.csv`. The `default-party.csv` files were likely a legacy pattern from before the unified manifest existed.

### Files to Reference

| File | Purpose |
| ---- | ------- |
| `_bmad/bmm/teams/default-party.csv` | BMM module roster (15 agents) — to be removed |
| `_bmad/tea/teams/default-party.csv` | TEA module roster (1 agent) — to be removed |
| `_bmad/cis/teams/default-party.csv` | CIS module roster (12 agents) — to be removed |
| `.claude/skills/bmad-party-mode/steps/step-01-agent-loading.md` | Party-mode loading logic — confirm reads agent-manifest.csv |
| `_bmad/_config/agent-manifest.csv` | Canonical agent manifest — party-mode's actual data source |

### Technical Decisions

1. **Remove, don't repurpose** — dead code should be removed; if Team Factory needs team templates, design them fresh with proper schema
2. **Verify before delete** — grep for any `default-party` references in code, workflows, and install scripts
3. **No migration needed** — these files are not generated by the installer, so removal doesn't affect the update system
