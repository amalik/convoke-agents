## Challenges and Growth Areas

1. **Subprocess testing learning curve** — Story 5.1 initially assumed in-process mocking would work for CLI testing. 13 `process.exit()` calls forced rethinking to subprocess approach via `runScript` helper. Discovery cost upfront time but paid off in 5.2.

2. **empathy-map naming collision** — `empathy-map` directory serves dual purpose as legacy v1.0.0 detection marker AND current Isla workflow name. Caused subtle test setup failures. Lesson carried from 5.1 to 5.2 successfully.

3. **Consistent ~5 findings per story in code review** — Content stories produced same error density as code stories. Dev agent doesn't self-catch factual inaccuracies, specificity loss, or visibility issues before review. Accepted as healthy process cost, not a defect rate.

4. **Specificity loss in content** — Story 5.4 lost concrete details (sprint estimates, timing) during content extraction. Story 5.5 had options hidden in HTML comments. Pattern: concrete details get stripped or hidden during initial authoring, caught in review.

5. **NFR11 jargon elimination is ongoing discipline** — Required sustained effort across all 3 content stories. HC ID references, technical vocabulary, implementation jargon all needed systematic removal. Not a one-time fix but a continuous practice.

---

## Key Insights and Learnings

1. **Adversarial review is a quality escalator, not overhead** — The ~5 findings/story rate is the cost of quality. Zero findings escaped to production. The process is catching everything. Natural friction of good process.

2. **NFR11 jargon elimination requires deliberate attention** — Each content story needed explicit passes to remove HC IDs, "handoff contract" references, "manifest" vocabulary. Must be an active checklist item, not assumed.

3. **Content stories need full code review** — Same error density as code stories. Editorial review catches prose/structural issues; adversarial review catches factual accuracy. Both necessary (lesson reinforced from Epic 4).

4. **Design-first approach reduces downstream iteration** — 5.3's design spike meant 5.4 and 5.5 had fewer design-related findings and moved faster.

5. **Lesson carry-forward compounds across stories** — When 5.1 discoveries flow into 5.2, and 5.3 patterns cascade through 5.4-5.5, each story starts from a higher baseline.
