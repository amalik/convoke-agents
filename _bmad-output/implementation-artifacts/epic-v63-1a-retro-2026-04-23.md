## What Went Well

### 1. The convergence rule held under pressure
Every story that had HIGH findings in Round 1 triggered Round 2 automatically. Round 2 caught *two separate regressions introduced by Round 1 patches* (1A.1 count arithmetic, 1A.2 `projectRoot='/'`). This is the rule earning its keep — without Round 2, those bugs would have shipped.

### 2. Decisions never cascaded to Round 3
Across the epic, every `decision-needed` finding was resolved cleanly in Round 1 or Round 2. The option-menu format ("1/2/3/4") consistently produced aligned operator decisions. D1 on 1A.6 (strip the custom-skill section referencing nonexistent machinery) was particularly high-leverage — shipped 1A.6 without an Epic-2 dependency.

### 3. Pattern reuse across stories accelerated development
- **Atomic write** (tmpfile + rename, optionally `crypto.randomBytes` + fsync) was invented for 1A.4 Phase 3 state writes, carried forward to 1A.6 CSV writes, then hardened again in v63-2-1.
- **`scripts/audit/` module skeleton** (1A.3's `audit-bmad-init-refs.js`) became the template for v63-2-1's `audit-bmm-dependencies.js` — ~30% development acceleration from copying the shape.
- **`_internal` export pattern for testability** repeated across 1A.2, 1A.3, 1A.4, 1A.6 code — now de facto Pattern 1 convention.

### 4. Scope discipline held
No story broke its "no changes to" boundary list (except one caught-at-review case — 1A.6 checklist.md scope-leak was identified and reverted in D2). Every story shipped the promised deliverable without bleeding into adjacent stories' scope.

### 5. Parallel adversarial review worked when it worked
Blind Hunter / Edge Case Hunter / Acceptance Auditor in parallel consistently surfaced non-overlapping findings. The 3-reviewer coverage caught 23 findings on 1A.4 alone — one reviewer would have missed at least half.

## What Didn't Go Well

### 1. LOC projections consistently undersized
- 1A.2: projected ~40–50 LOC loader → shipped 285 LOC after Round 1 hardening (5.5× projection).
- 1A.4: projected 250–350 LOC → shipped 570+ LOC after Round 1 patches.
- 1A.6: projected single-page guide → fine on size, but hit 3 decisions + 5 patches in review.
- v63-2-1 (next epic carryover): projected 250–350 → shipped 420.

**Root cause:** projections baseline the happy path. Defensive hardening (path validation, error message polish, edge-case coverage) consistently adds 40–80% LOC.

### 2. Subagent API instability derailed 1A.3 Round 1
When all three subagent reviewers failed to launch, the review pivoted to a single inline pass. Coverage was still adequate (no HIGH findings), but the layered-adversarial property was lost. No graceful-degradation pattern was established for this failure mode.

### 3. Round 1 patches introduced new HIGHs in 2 of 6 stories
- 1A.1: count arithmetic regression (E14 cross-references double-counted).
- 1A.2: trailing-slash normalization introduced `projectRoot='/'` silent cwd-load bug.
- 1A.6: M7 CRLF normalization mutated CRLF inside quoted fields (M7 × H2 interaction).

The convergence rule caught all three in Round 2 — but the pattern says: "when a patch introduces a normalization or guard clause, test the extremes."

### 4. Story dates and cross-references drifted
Multiple stories had spec-drift issues (1A.1's "14 bme agents / 26 behaviors / 4/10/12" stale counts; 1A.4's `preview(projectRoot)` spec vs runtime `preview()`; 1A.6's `scripts/convoke-update.js` vs `scripts/update/convoke-update.js`). Each caught during review, but they accumulated because no preflight validated story-spec references against the code.

**Opportunity:** `spec-verify-referenced-files` rule was invoked multiple times at authoring — but not at review-start. Adding it as a pre-review gate would pre-empt these.

### 5. The DoD checklist churned twice during the epic
- lint-1.1 (cross-cutting) tightened the lint gate to unconditional `required-inputs`.
- 1A.6 attempted to loosen it again with `conditional-inputs` for pure-docs stories.
- 1A.6 Round 1 D2 reverted.

**Lesson:** process changes that loosen a just-tightened gate should trigger their own story, not ride along as a scope leak.

- **1A.4's `D2 parallel registry entries`** — the insight that 3.0/3.1/3.2 users need explicit parallel `-to-4.0.0` migrations (not just chain-walking through older minor versions). Unlocked the v1.7.x multi-hop test coverage in 1A.6 R2 verification.
- **1A.6 D1 strip decision** — the realization that shipping a guide referencing machinery Epic 2 hasn't built yet would *teach users to ignore future correct instructions*. Chose strip over "coming soon disclaimer" — an honest-software call.
- **v63-2-1 H3 self-reference filter** (immediately downstream) — the same restraint applied: filter out `references/` doc examples so the CSV registry reflects reality, not aspirational detection.
- **Pattern: "documented process claim > overpromised future feature"** — 1A.6 D3 rephrased PF1 forward-promise to present-tense-about-process. The guide will age gracefully even if Epic 4 slips.

## Key Lessons (bite-sized, for reuse)

1. **Convergence rule pays for itself.** Every HIGH finding → Round 2 → regressions caught. Never shortcut.
2. **Strip > defer > aspiration.** When shipping docs that reference unshipped machinery, strip the reference. Don't ship "coming soon."
3. **LOC projections are happy-path only.** Budget 60% overhead. Treat Round 1 LOC expansion as expected, not surprising.
4. **Pattern reuse compounds.** The audit-tool shape shipped three times; the atomic-write pattern shipped twice; `_internal` export pattern became convention. Invest in the second occurrence being cleaner than the first.
5. **Normalization/guard patches need extreme-value tests.** Two Round 2 regressions traced to this. Build it into Round 2 prompts.
6. **Scope discipline survives review — if the reviewer is watching for scope leaks.** 1A.6 D2 was caught only because an auditor explicitly flagged "this file isn't in the story's File List." Without that check, the regression would have shipped.
7. **Parallel adversarial review is not optional when API is stable.** Non-overlapping coverage is the value; fall back only under explicit failure.
